# ラジオ番組開始（まくら）シナリオ生成プロンプト (1人用)

## 概要
このプロンプトは、ラジオ番組の開始部分（まくら）のトークシナリオを生成するために使用します。
パーソナリティ（1人）による自然な番組導入を作成します。

---

## プロンプト本文

```
あなたは、ラジオ番組の台本作家です。
以下の条件で、番組開始部分（まくら）のトークシナリオを作成してください。

【登場人物】
- パーソナリティ（MC）: 明るく元気な性格で、リスナーに向けて親しみやすく一人語りで番組を進行

【番組コンセプト】
- デスクトップマスコットによるインタラクティブなラジオ番組
- リスナー（ユーザー）との対話を重視
- カジュアルで親しみやすい雰囲気

【シナリオの条件】
1. トークは4〜6ブロック程度（文脈の区切りごとに配列に分割）
2. 一人語りとしての自然な語り口、テンポとリズムを保つ
3. 番組の雰囲気作りを意識した内容
4. リスナーへの挨拶と番組紹介を含める
5. 今日のテーマや企画の予告について軽く触れる

【出力形式】
以下のJSON形式で出力してください：

{
  "scenario": "opening",
  "title": "番組開始",
  "conversations": [
    {
      "speaker": "MC",
      "text": "発言内容",
      "emotion": "happy/neutral/excited"
    }
  ]
}

【注意事項】
- 各ブロックの発言は1〜3文程度の自然な長さにしてください
- 音声合成に適した読みやすい文章にしてください
- 感情表現（笑い、驚き等）は emotion フィールドで表現してください
- セリフに（笑）などの地の文は含めないでください
```

---

## カスタマイズオプション

### 時間帯別バリエーション
朝・昼・夜で異なる挨拶を使いたい場合：

```
【追加条件】
- 現在の時間帯: {morning/afternoon/evening/night}
- 時間帯に応じた挨拶を使用してください
```

### キャラクター性の調整
特定のキャラクター設定を反映する場合：

```
【キャラクター設定】
- MC: {性格、口調、特徴}
```

## 実装上の注意

1. **感情フィールドの活用**
   - StyleBertVits2のスタイル制御に使用
   - 適切な感情を選択することで、より自然な音声表現が可能

2. **発言の長さ**
   - 1ブロックの発言は100文字以内を推奨
   - 長すぎると音声合成に時間がかかる

3. **テンポ**
   - ひとり語りであるため、リスナーに語りかけるような適度な間を意識したテキスト構成にする
   - 一方的な情報の詰め込みにならないように注意

4. **リスナーへの配慮**
   - リスナーの存在を意識した発言（「〜ですか？」などの問いかけ）を含める
   - 対話型番組であることを強調
