# アプリケーション再起動機能の実装計画 (更新2)

開発環境において `app.relaunch()` を実行した際、`vite-plugin-electron` が古いプロセスの終了を検知して監視・クリーンアップ処理を実行するため、親プロセスごとクラッシュする現象（`ERROR: The process "XXXX" not found.`）が発生します。

本番パッケージング環境では問題なく動作しますが、開発環境での利便性とクラッシュ防止のため、開発環境での再起動実行時における挙動を調整します。

## 提案する変更内容

### Electron メインプロセス

#### [MODIFY] [main.ts](file:///c:/workspace/workspace-win/DesktopAiMascot/ui/electron/main.ts)
- `relaunch-app` メッセージリスナーにおいて、開発環境（`process.env.VITE_DEV_SERVER_URL` が存在する場合）では `app.relaunch()` を行わず、ダイアログ（`dialog.showMessageBox`）で「開発環境では自動再起動できません。手動で `npm run dev` を再実行してください。」といったメッセージを表示し、アプリを終了（`app.quit()`）させるか、再起動処理をキャンセルできるようにします。
- 本番環境では従来どおり自動的に `app.relaunch()` と `app.exit(0)` を実行します。

## 検証計画

### 開発環境での手動検証
1. `npm run dev` でアプリを起動します。
2. 設定画面の「再起動」ボタンをクリックします。
3. 開発環境である旨のダイアログが表示されることを確認します。
4. キャンセルした場合は何も起きず、OKを押した場合はアプリが安全に終了することを確認します（親プロセスがクラッシュするエラーが発生しなくなります）。
