# 修正内容の確認 (Walkthrough) - ラジオモードでの能動的発話（アクティブトーク）

ラジオモードがONの際、ユーザーが何も話しかけていない沈黙状態でも、マスコットから能動的に話題を切り出したりフリートークを開始したりする機能を実装しました。

---

## 変更内容の概要

### 1. メッセージ送信処理の拡張
- [useChatConnection.ts](file:///c:/workspace/workspace-win/DesktopAiMascot/ui/src/components/chatpanel/useChatConnection.ts)
  - `sendMessage` メソッドを拡張し、引数 `isActiveTalk: boolean` を受け取れるようにしました。
  - `isActiveTalk` が `true` の場合、ユーザー側のメッセージ入力バリデーションをスキップし、チャットUI上のユーザー吹き出しを生成せずにLLMにリクエストを送るようにしました。
  - システムプロンプトを「沈黙を破ってラジオ番組のフリートークを開始する」「自発的に新しい話題（季節、天気、雑談、問いかけ）を切り出す」ように制御するプロンプトを動的に差し込みます。

### 2. チャットUIでのタイマー管理
- [ChatPanel.vue](file:///c:/workspace/workspace-win/DesktopAiMascot/ui/src/components/ChatPanel.vue)
  - 30秒（デフォルト設定値）の自動おしゃべりタイマーを実装しました。
  - タイマーは以下の状態変化のたびに適切にクリア・再始動されます：
    - ユーザーが文字を入力し始めた時（キー入力中はタイマー一時停止）
    - マスコットが返答中・音声読み上げ中（発話中はタイマー一時停止）
    - ラジオモードのON/OFF切り替え時

---

## 検証方法（手動テスト）

アプリ起動後、以下の動作を確認してください。

1. **ラジオモードのON**:
   - チャットヘッダーのマイクボタンを押してラジオモードをONにします。
2. **自動発話（能動トーク）の確認**:
   - そのまま何も入力・送信せずに **30秒間** 待機します。
   - チャット履歴にユーザー側の発言を追加することなく、マスコット側の吹き出しだけが追加され、「さて、次の話題ですが…」「リスナーの皆さん、最近はどうお過ごしですか？」といった自発的なフリートークや問いかけ（チャット文＋音声）が開始されることを確認します。
3. **入力・発話中の割り込み防止**:
   - ユーザーが文字を入力し始めた時、あるいはマスコットが音声を発声している間は自動トークのタイマーがリセットされ、勝手に自動トークが発火しないことを確認します。
