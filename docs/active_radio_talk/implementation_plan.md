# ラジオモードでの能動的発話（アクティブトーク）機能の実装計画

ラジオモードがONの際、ユーザーが話しかけなくても、マスコット側から定期的に能動的に新しい話題を振ったりフリートークを展開したりする機能を実装します。

## ユーザーレビュー要求事項

> [!NOTE]
> **タイマーの動作仕様**
> - 能動発話タイマーのデフォルトの待機時間は **30秒** とします。
> - ユーザーがキー入力中（メッセージ入力欄が空でない場合）や、マスコットが返答・音声再生中の間は、タイマーが自動でリセット・一時停止します。
> - ユーザーの吹き出しをチャット画面に表示せずに、マスコットの吹き出しだけが一方的に増えていくラジオ番組風の会話体験を構築します。

## 変更内容

### 1. マスコット接続管理の拡張
#### [MODIFY] [useChatConnection.ts](file:///c:/workspace/workspace-win/DesktopAiMascot/ui/src/components/chatpanel/useChatConnection.ts)
- 自発的なトークをLLMにリクエストする `sendActiveTalkMessage()` メソッドを追加します。
  - このメソッドでは、ユーザーメッセージ履歴として「(沈黙...)」というメタデータをLLMに渡しつつ、システムプロンプトに「会話が途切れたので、ラジオ番組の進行として新しい話題（フリートーク、季節の話題、リスナーへの問いかけなど）を切り出してください」という特別指示を追加して送信します。
  - チャットUIのメッセージ一覧 (`messages.value`) には、ユーザーの吹き出しを追加せず、マスコットの「考え中...」および応答だけを追加します。

### 2. チャットUIでのタイマー管理
#### [MODIFY] [ChatPanel.vue](file:///c:/workspace/workspace-win/DesktopAiMascot/ui/src/components/ChatPanel.vue)
- ラジオモードのON/OFFおよびマスコットの発話状態 (`isSpeaking`, `isAiResponding`)、ユーザーの入力状態 (`inputText`) を監視するタイマー管理ロジックを実装します。
- `isRadioMode` が ON の間、一定時間（30秒）何もない場合に `sendActiveTalkMessage()` を呼び出します。
- 以下の条件の時にタイマーをクリア・リセットします：
  - ユーザーが手動でメッセージを送信した時
  - マスコットの応答・音声合成の読み上げが完了した時 (`isSpeaking` が `false` になった時)
  - ユーザーがメッセージ入力中の時 (`inputText` が空でない時)

## 検証計画

### 手動検証
1. **ラジオモードのON**:
   - マイクアイコンをクリックしてラジオモードをONにします。
2. **自動発話の確認**:
   - 30秒間、何も入力せずに待機します。
   - マスコット側から「さて、次の話題ですが…」「リスナーの皆さん、最近はどうお過ごしですか？」といった能動的な発話（チャット表示＋音声）が自動的に開始されることを確認します。
3. **割り込み動作の確認**:
   - マスコットが自動トーク中、または手動でテキストを入力し始めた際にタイマーが適切にリセットされ、自動トークが重複して発生しないことを確認します。
