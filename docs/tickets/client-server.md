# クライアント＆サーバ化の設計と実装

## 概要

現在単一のデスクトップアプリケーション（Electron + Vue3）として密結合で動作しているデスクトップAIマスコットを、マスコット描画やチャットUIを担う「クライアント」と、AI推論・音声合成連携・設定データおよびキャラクター状態管理（感情や記憶）を一元的に行う「サーバー」へとアーキテクチャを分離・構成変更する。

## 目的

- **クライアントの軽量化:** 常駐するマスコット（クライアント）のメモリ・CPU使用率を最小限に抑え、PC操作を邪魔しない極限の軽量化を実現する。
- **処理のオフロード:** 重いAI推論（ローカルLLM）や音声合成（VOICEVOX等）の通信処理、およびそのオーケストレーションをバックエンド（サーバー）側に集約する。
- **アセットと設定の一元管理:** 立ち絵、表情、服装などのキャラクターアセットをサーバー側で集中管理・ロードし、クライアントへストリーミングやオンデマンドで配信可能にする。
- **マルチクライアント対応:** 単一のAIサーバーに対して、デスクトップマスコットのほか、スマートフォン、Webブラウザなど複数のクライアントから同一のキャラクター（コンテキストや記憶）にアクセス・連携できる基盤を構築する。

## 完了条件

- **設計書の作成:** クライアント/サーバー間の通信方式（WebSocket, HTTP/REST API, または gRPC）およびシステム構成図が docs 内に作成されていること。
- **通信API仕様の策定:** チャット対話、感情変化同期、音声ストリーミング、アセット配信のための各種APIエンドポイントが定義されていること。
- **プロトタイプの実装:**
  - AI推論および音声合成への接続をサーバー側に隠蔽し、クライアントはサーバーAPI経由でやり取りする構造になっていること。
  - クライアントがサーバーのアドレスとポートを指定して動的に接続・データ同期を行えること。
- **テスト・動作検証:** ローカル環境、およびローカルネットワーク上の別端末からの接続テストが正常に通り、マスコットの挙動（アニメーション、発話、感情反映）が損なわれていないこと。
