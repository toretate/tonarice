# Image-to-Image (i2i) モード実装仕様書

## 1. 概要
本仕様書は、デスクトップマスコットのチャット画面において、従来のテキストからの画像生成（t2i）機能に加え、添付された画像をベースに新しい画像を生成・編集する Image-to-Image（i2i）機能を追加するための仕様を定義するものです。

---

## 2. 機能要件

### 2.1. 画像生成・編集パラメータの追加
- Stable Diffusion WebUI Forge API へのリクエストパラメータに以下を追加します。
    - **ノイズ除去強度 (Denoising Strength)**
        - パラメータ名: `forgeDenoisingStrength` (内部 API には `denoisingStrength` として受け渡し)
        - 型: `number`
        - 設定範囲: `0.0` ～ `1.0` (デフォルト: `0.70`, ステップ: `0.05`)
        - 用途: i2i 生成時、元画像からどの程度変化させるかを制御する。
        - **表示位置変更**: i2i モード時は重要な調整項目となるため、設定ダイアログ内ではなく、チャット入力欄の上部インジケーター内にリアルタイム変更可能なゲージ（スライダー）として表示します。
    - **ベースプロンプト (Prompt)**
        - パラメータ名: `forgePrompt`
        - 型: `string`
        - 用途: 生成時に常に適用する基本プロンプト。
    - **ネガティブプロンプト (Negative Prompt)**
        - パラメータ名: `forgeNegativePrompt`
        - 型: `string`
        - 用途: 生成時に除外したい要素。
        - **適用ルール**: エンジン毎に適用可否が異なるため、ローカル SD 系エンジン（`sd_forge` など）が選択されている場合にのみ入力欄を表示し、生成時も SD 系のみに適用します（DALL-E 3 等では無効化または非表示）。
    - **サンプラー (Sampler)**
        - パラメータ名: `forgeSampler`
        - 型: `string` (デフォルト: `Euler a`)
        - 用途: 画像生成時のサンプリングアルゴリズムを指定します（SD 系のみ適用）。

### 2.2. パラメータ設定ダイアログ (`ForgeImageGeneratorDialog.vue`) の項目配置と表示ルール
設定ダイアログのコントロールは以下の順序で配置し、選択されている画像生成エンジン（SD系か否か）によって表示を動的に制御します。

1. **画像生成エンジン (共通)**: `localEngine` (dalle3, sd_forge 等)
2. **使用モデル (SD系のみ)**: `localModel`
3. **使用 LoRA (SD系のみ - 複数追加および個別の適用強度調整)**:
   - 各 LoRA ごとに適用強度 (Weight / Strength、範囲 `-2.0` ～ `2.0`、デフォルト `1.00`) を個別に設定できます。
   - UI上では、LoRAの追加ボタンおよび、追加されたLoRA一覧（ドロップダウンでの名称選択、強度調整スライダー、削除ボタン）をリスト形式で表示します。
   - **フォルダフィルタ機能**: 「LoRAを追加」ボタンの横に、LoRAのサブフォルダ（例: `xl/style/`）で絞り込むためのフォルダフィルタ用ドロップダウンを配置します。フォルダが選択されると、各 LoRA 行のドロップダウン候補がそのフォルダ内の LoRA に限定されます。
   - ストアでは `lora1:weight1, lora2:weight2` のようにカンマ区切りおよびコロン区切りのフォーマット文字列で `forgeLora` に保存し、生成時に `<lora:LoRA名:適用強度>` として自動的にプロンプト末尾へ連結します。
4. **Prompt (共通)**: `localPrompt` (テキストエリア)
5. **Negative Prompt (SD系のみ)**: `localNegativePrompt` (テキストエリア)
6. **画像解像度 (共通)**: `resolution` (1024x1024 等)
7. **Sampler (SD系のみ)**: `localSampler` (Euler a, DPM++ 2M Karras 等のドロップダウン)
8. **Sampling Steps (SD系のみ)**: `localSteps` (スライダー)
9. **CFG Scale (SD系のみ)**: `localCfgScale` (スライダー)

### 2.2.A. 生成設定プリセット機能
設定ダイアログの最上部に、すべての設定値（エンジン、モデル、LoRA構成、プロンプト、解像度、Steps、CFGなど）を名前付きで「プリセット」として保存・読み込み・削除できる機能を配置します。
- **データ永続化**: `configStore` 内に `forgePresets` (JSON化された `ImageGenPreset` の配列文字列) を保持・永続化します。
- **UIコンポーネント**: プリセット一覧選択用のドロップダウン、新規プリセット名入力テキストボックス、保存ボタン、削除ボタンを最上部に配置します。

### 2.3. 画像モード選択メニューの追加
- チャット画面ヘッダーにある「画像生成」ボタン（従来の `isT2iMode` トグル）の挙動を変更します。
- ボタンをクリックすると、ドロップダウンメニューが表示され、以下の項目を選択可能にします。
    1. **テキストから画像生成 (t2i)**: `imageGenMode = 't2i'` に設定
    2. **画像から画像生成 (i2i)**: `imageGenMode = 'i2i'` に設定
    3. **生成パラメータ設定**: `ForgeImageGeneratorDialog` ダイアログを表示

### 2.4. チャット入力エリアとの統合 (t2i / i2i モード)
- `imageGenMode`（値は `'t2i' | 'i2i' | null`）の状態を定義し、画像生成・編集モードを管理します。
- **モード共通仕様**:
    - 通常のチャット送信ではなく、画像生成・編集のプロンプト入力として機能します。
    - チャット入力フォームの上に、現在どの画像生成・編集モードが有効であるかを示すインジケーターを表示し、「チャットに戻る」ボタンで即座に解除できるようにします。
- **i2i モード固有仕様**:
    - チャット欄のプレースホルダーを `[画像編集 (i2i) モード] 編集の指示を入力...（元画像が必要です）` に変更します。
    - 送信時に `pendingAttachments` (ユーザーがアップロードした添付ファイル一覧) のうち、最初の画像ファイルを元画像（`initImage`）として使用します。
    - 送信時に画像が添付されていない場合は警告（`alert` またはエラー表示）を出し、生成リクエストを送信しません。
    - 生成が正常に完了、または失敗した段階で、添付されていた元画像をクリアします。

## 3. UI/UX デザイン要件
- **ドロップダウンメニュー**:
    - ヘッダーの画像ボタン下に表示されるフローティングメニュー。グラスモーフィズムに調和する背景と影を持ち、現在選択されているモードにチェックマーク (pi-check) を表示します。
- **画像モードインジケーター**:
    - 入力フォーム上部に、薄いパープル系の背景色 (`rgba(168, 85, 247, 0.1)`) と境界線を持ち、選択中のモードを明示するバッジと「チャットに戻る」ボタンを配置します。
- **レスポンシブ / 配置**:
    - コンパクトモードやスプリットモードのウィンドウ幅に収まるよう、ドロップダウンメニューやインジケーターのスタイルを適切に調整します。
