# 音楽再生ウィジェット仕様

## 1. 目的

ローカル端末上の音楽フォルダを選択し、フォルダ内の対応音源を再生できる音楽ウィジェットを提供します。本仕様は GitHub Issue #32 のローカル音楽再生を対象とします。

## 2. 対応画面

- 統合モード: 画面下部へ横長ウィジェットとして固定表示します。ドラッグ移動は行いません。
- コンパクトモード: チャットの操作パネル直下へ「曲名／再生・停止／次の曲」の1行をオーバーレイ表示します。曲が未選択の場合は曲名部分からフォルダを選択できます。
- 分離モード: 縦長の専用音楽ウィンドウを開閉します。ヘッダーのドラッグで位置を変更できます。
- Web 版: 統合／コンパクトモードと同じフォルダ選択・再生機能を利用できます。

## 3. 対応機能

- 端末のフォルダ選択ダイアログから音楽フォルダを1つ選択し、フォルダ配下の対応音源をプレイリストへ読み込みます。別のフォルダを選択した場合はプレイリストを置き換えます。
- 対象拡張子は MP3、M4A、AAC、WAV、Ogg、FLAC、Opus、WebM です。実際に再生できるコーデックは Chromium の対応範囲に従います。
- 再生、一時停止、前の曲、次の曲、再生位置の変更、音量変更を行えます。
- シャッフル、リピートなし、全曲リピート、1曲リピートを切り替えられます。
- プレイリストは展開／折り畳みを切り替えられます。展開時は曲の選択、個別削除、全削除を行えます。
- `アーティスト - 曲名.ext` 形式のファイル名は、アーティスト名と曲名へ分けて表示します。それ以外は拡張子を除いたファイル名を曲名として表示します。
- Media Session API が利用可能な環境では、OS のメディアキーから再生、一時停止、前後の曲を操作できます。

## 4. 永続化とプライバシー

- 表示状態、音量、シャッフル、リピートモード、透明度、プレイリストの展開状態を `localStorage` に保存します。
- 選択したファイル、ファイルパス、プレイリストは保存しません。アプリを再読み込みした場合は再選択が必要です。
- 音声データは外部サービスやサーバーへ送信せず、レンダラー内の Object URL で再生します。曲を削除したときとウィジェット破棄時に Object URL を解放します。

## 5. 設定画面

- 設定画面のサイドバーに「音楽ウィジェット」を表示します。
- ウィジェットの歯車ボタンから音楽ウィジェット設定を直接開けます。
- 透明度（20～100%）、初期音量、プレイリストの展開状態を変更できます。
- 設定変更は同一オリジンの各ウィンドウへ `storage` イベントで同期します。

## 6. iTunes 連携

iTunes／Apple Music 固有のライブラリ参照や再生制御は初期実装の対象外です。Windows の iTunes COM、macOS の Music.app、Apple Music API は認証方式と提供機能が異なるため、クロスプラットフォーム要件、認証、ライブラリ同期方式を別 Issue で確定してから追加します。

## 7. 受け入れ条件

1. チャットヘッダーから各ウィンドウモードの音楽ウィジェットを開閉できます。
2. ローカル音楽フォルダを選択し、基本操作、シーク、音量、シャッフル、リピートを操作できます。
3. 曲の削除またはウィジェットの破棄で、作成した Object URL が解放されます。
4. 音楽ファイルやローカルファイルパスが永続化・外部送信されません。
5. 統合モードは画面下部の横長固定、分離モードは移動可能な縦長、コンパクトモードは操作パネル直下の1行表示になります。
6. 設定画面から透明度、初期音量、プレイリスト展開状態を変更できます。
7. ストア、モード別表示、曲名解析、曲順計算、時間表示のユニットテストが成功します。
