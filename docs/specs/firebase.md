# Firebase 導入検討仕様書

本ドキュメントでは、デスクトップAIマスコットアプリにおける Firebase 導入のメリットとデメリットについてまとめます。

---

## 1. Firebase 導入のメリット

### 1.1 ユーザー認証の容易な統合 (Firebase Authentication)
* **概要**: Google、GitHub、メールアドレス等によるログイン機能を迅速に構築できます。
* **メリット**:
    * 複数デバイス（PC版と Android 版）の間で同一のユーザーアカウントを使用してマスコットの設定や対話履歴を共有できます。
    * 認証フローの安全な管理を Firebase 側に委ねることができます。

### 1.2 リアルタイムなデータ同期とクラウド保存 (Cloud Firestore)
* **概要**: NoSQL クラウドデータベースにより、ユーザー設定やマスコットのステータスをクラウドに保存します。
* **メリット**:
    * 設定の変更やマスコットの成長ステータスが即座に同期されます。
    * ローカルデータが破損・紛失した場合でも、クラウドからデータを復元できます。
    * オフラインキャッシュ機能が組み込まれているため、一時的なオフライン状態でも動作を継続できます。

### 1.3 アプリケーション設定の動的更新 (Firebase Remote Config)
* **概要**: アプリのアップデートを行うことなく、クラウドから設定パラメータを変更します。
* **メリット**:
    * マスコットの対話プロンプト、アニメーション速度、イベント発生確率などのバランス調整をアプリのビルドなしで即座に反映できます。
    * 特定のユーザー層に向けた機能の段階的リリース（A/Bテスト）が可能です。

### 1.4 プッシュ通知の送信 (Firebase Cloud Messaging - FCM)
* **概要**: クロスプラットフォーム対応のメッセージ送信機能です。
* **メリット**:
    * Android デバイスや PC（Webプラットフォーム等）に対して、バックグラウンド状態でもマスコットからの呼びかけや Antigravity のイベント通知を送信できます。

### 1.5 セキュアなバックエンド処理の隠蔽 (Cloud Functions for Firebase)
* **概要**: サーバーレスでサーバーサイドロジックを実行します。
* **メリット**:
    * Gemini API やその他の外部 AI サービスの API キーをクライアントアプリ内に含めることなく、安全にバックエンド経由で呼び出すことができます。

## 2. Firebase 導入のデメリット

### 2.1 デスクトップ環境における SDK サポートの制限
* **概要**: Firebase は元々モバイル (iOS/Android) および Web をメインターゲットとして設計されています。
* **デメリット**:
    * アプリが C# (.NET / Windows Forms / WPF) で構築されている場合、公式の Firebase SDK は完全なサポートを提供していない場合があります（一部のサービスでは C++ SDK のラッパーやサードパーティ製ライブラリ、または REST API 経由でのアクセスが必要になります）。
    * フロントエンドが Web 技術（Vite / Electron / Tauri）で構築されている場合は Web SDK を流用できますが、デスクトップ固有の考慮事項が発生することがあります。

### 2.2 常時オンライン接続への依存
* **概要**: クラウド主体の設計であるため、ネットワーク接続状態に依存します。
* **デメリット**:
    * 完全なスタンドアロン（オフライン）環境での動作を前提とした場合、Firebase に依存した設計は動作制限やエラーハンドリングの複雑化を招きます。

### 2.3 コストと利用制限（ベンダーロックイン）
* **概要**: 無料枠（Spark プラン）を超えた場合の従量課金システムです。
* **デメリット**:
    * アプリの利用者数が増加したり、対話履歴などのテキストデータを大量に読み書きするようになると、維持コスト（Blaze プラン）が急激に上昇するリスクがあります。
    * 他のクラウドプラットフォーム（Supabase、AWS、自前サーバー等）へ移行する際、データベースのデータ構造や認証ロジックの移行コストが非常に高くなります。

### 2.4 クライアントアプリのサイズ増加
* **概要**: SDK の追加によるバイナリ容量の増加です。
* **デメリット**:
    * アプリケーションのパッケージサイズが増加し、起動時間やメモリ使用量に影響を与える可能性があります。
