# タスク概要: ExpressバックエンドとElectronフロントエンドのNuxt統合

## 目的
本タスクの目的は、Expressで記述された独立したバックエンドサーバー（`server`）と、Electron/Vue 3で構成されたフロントエンド（`ui`）を、単一の **Nuxt** プロジェクトに統合することです。

これにより、以下のメリットを実現します：
- **サーバー管理の簡素化**: バックエンドロジックを Nuxt Nitro サーバーに集約し、Express とフロントエンドの二重管理を解消。
- **純粋な Server-Client モデルの確立**: IPC 通信による限定された連携から、同一の HTTP/WebSocket ポートを通した通信モデルへの完全移行。
- **単一プロセスでの実行可能性**: Electron 起動時に Nitro サーバープロセス（`.output/server/index.mjs`）を自動的にスポーン・管理し、ユーザーの運用負荷を低減。

## 成果基準
1. **バックエンド機能の移行完了**:
   - `server` 側の各 API ルートが `ui/src/server/api` および `/server/routes` 下の Nitro ルートとして動作する。
   - バックエンドで使用していた音声合成や AI チャット、背景除去などの重い処理が、Nuxt の Nitro サーバー上で問題なく処理できる。
2. **WebSocket 通信の動作**:
   - Nitro サーバ側で WebSocket ルート（`ws.ts`）が定義され、以前のチャット双方向通信が同じロジックで再現できる。
3. **Electron による自動管理**:
   - 本番用 Electron 起動時に空きポートを自動検出して Nitro サーバーを起動する。
   - アプリケーション終了時に Nitro サーバープロセスもクリーンに強制終了（SIGTERM）される。
4. **ドキュメントの完備**:
   - 移行の計画と変更点、確認方法が `docs/nuxt_migration/` 以下にドキュメントとして格納されている。
