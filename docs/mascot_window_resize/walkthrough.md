# 修正内容の確認 (Walkthrough)

マスコットウィンドウおよびキャラクター自体のサイズを変更できる機能を実装しました。これにより、デスクトップ上での存在感や視認性をユーザーの好みに合わせて柔軟にカスタマイズできるようになります。

### 🌟 標準サイズ（100%）基準の 1.5 倍化調整 (追記)
ご要望に基づき、以前の「大（150%）」のサイズを、新しい「標準（100%）」として再定義・アジャストしました。
*   **ウィンドウサイズの引き上げ**: ベースのウィンドウサイズ定数を `300x400` から `450x600`（1.5倍）へ引き上げました。
*   **描画ベーススケールの拡大**: キャラクター描画サイズを `180px` から `270px`、ベースフォントサイズを `110px` から `165px` へ拡大しました。
*   **表情レイヤーの座標調整**: 重ね合わせる表情画像の基準サイズを `60px` から `90px`、基準 Y 軸オフセットを `-8px` から `-12px`、絵文字表示の表情の位置補正 `translateY` を `-10px` から `-15px` へ拡大し、表情エディタの位置パラメータ換算比率 `scaleFactor` を `270 / 420` へ更新しました。

これらにより、以前の「大」の大きさが新しい「標準（100%）」となり、そこからさらに `0.5` 〜 `2.0` 倍へとシームレスかつ綺麗にリサイズが可能となっています。

## 変更内容

### 1. バックエンド (Electron) の拡張
*   **[main.ts](file:///c:/workspace/workspace-win/DesktopAiMascot/electron/main.ts)**:
    *   設定データに `mascotScale: number` (初期値 `1.0`) を追加し、起動時にこのスケールに基づいてマスコットウィンドウの初期サイズ (`300 * scale`, `400 * scale`) とチャットウィンドウの初期追従オフセット (`300 * scale`) を算出・適用するように実装しました。
    *   IPCイベント `set-mascot-scale` を新規追加し、設定の更新・ウィンドウサイズの動的変更 (`win.setSize`)・チャット位置の再配置、そして全ウィンドウへの最新設定データのブロードキャストを実行する処理を実装しました。
*   **[preload.ts](file:///c:/workspace/workspace-win/DesktopAiMascot/electron/preload.ts)**:
    *   レンダラープロセス向けAPIブリッジ `electronAPI` に `setMascotScale(scale)` メソッドを公開しました。

### 2. フロントエンド (Vue 3 / Pinia) の拡張
*   **[electron.d.ts](file:///c:/workspace/workspace-win/DesktopAiMascot/src/electron.d.ts)**:
    *   型安全性のために `IElectronAPI` に `setMascotScale` メソッドの型定義を追加しました。
*   **[config.ts (Pinia)](file:///c:/workspace/workspace-win/DesktopAiMascot/src/store/config.ts)**:
    *   ストアに `mascotScale` を追加し、設定ファイルの読み込み、localStorage バックアップ、および設定更新アクション (`updateConfig`, `saveConfig`) に統合しました。
*   **[MascotViewer.vue](file:///c:/workspace/workspace-win/DesktopAiMascot/src/components/MascotViewer.vue)**:
    *   マスコット要素である `.mascot-character` 全体に対して、ストアの `mascotScale` を適用した CSS の `transform: scale(...)` インラインスタイルを追加しました。これによって、キャラクター画像、表情レイヤー、絵文字サイズ、および吹き出しなどの要素が崩れることなく等倍で綺麗に拡大・縮小されます。
    *   滑らかなスケーリングを実現するため、CSS に `transform-origin: center` およびマイクロアニメーション効果 `transition: transform 0.1s ease-out` を追加しました。
    *   `Ctrl + マウスホイール` イベントを検知する `onWheel` リスナーを追加し、ホイールの回転方向に応じてスケールを `0.1` 刻みで変更（`0.5` 〜 `2.0` の範囲制限）し、メインプロセスにIPC送信するロジックを実装しました。
*   **[SettingsWindow.vue](file:///c:/workspace/workspace-win/DesktopAiMascot/src/components/settings/SettingsWindow.vue)**:
    *   サイドメニューの「チャットウィンドウ」タブの名称を「**ウィンドウ設定**」に変更し、UIの階層を整理して「マスコットウィンドウ設定」と「チャットウィンドウ設定」に分割しました。
    *   「マスコットウィンドウ設定」セクションに、`0.5`〜`2.0`（50%〜200%）の間で変更できるスライダーと、「小 (50%)」「中 (100%)」「大 (150%)」「特大 (200%)」を即座に選べるプリセットボタンを配置し、変更時に `window.electronAPI.setMascotScale` を呼び出すロジックを実装しました。

## 検証結果

### 1. コンパイル確認
*   `npm run build` によるビルドを実行し、TypeScript のコンパイルエラーやアセットのパッケージングにエラーがないことを確認しました。

### 2. 動作確認項目
*   **設定画面スライダー操作**: 「ウィンドウ設定」内のサイズスライダーを操作したとき、マスコットウィンドウと中のアバターがリアルタイムかつ滑らかに拡大・縮小すること。
*   **クイックサイズ変更ボタン**: 「小」「中」「大」「特大」のプリセットボタンをクリックした際、対応するサイズにスムーズに切り替わること。
*   **Ctrl + ホイール操作**: マスコット上で `Ctrl` キーを押しながらマウスホイールを回転させたとき、その場でシームレスにサイズが拡大・縮小すること。
*   **チャット位置の連動追従**: マスコットが拡大された際、チャットウィンドウのX軸追従位置がマスコットの新しい幅に自動適用され、重なりや隙間が生じないこと。
*   **設定永続化**: アプリケーションを再起動したときに、前回適用したサイズスケールが維持された状態で起動すること。
