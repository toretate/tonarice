# 修正内容の確認 (Walkthrough)：マルチユーザーマスコット管理

本ドキュメントは、マルチユーザー対応（内部ユーザーID仲介によるユーザーごとの設定・履歴・アセットの分離）の実装完了後に、動作確認を行うための手順書です。

## 1. 準備
1. サーバーを再起動します。
   ```bash
   # server ディレクトリで実行
   npm run dev
   ```
2. Electronアプリ（またはWebクライアント）を起動します。

---

## 2. 動作確認手順

### テスト項目 1: ローカル開発バイパスでの動作（初期化と保存）
ローカルホストから接続すると、自動的に内部ユーザーID `usr_local_dev_bypass` として扱われます。

1. **初期データの生成確認**:
   - アプリを起動し、何らかの設定変更（例: 「マスコットのスケール変更」や「ウィンドウの常に手前に表示の切り替え」など）を行い、保存します。
   - `server/users/usr_local_dev_bypass/` ディレクトリが自動作成されていることを確認します。
   - その中に `config.json` が生成され、変更した設定が保存されていることを確認します。

2. **チャット履歴の生成確認**:
   - マスコットとチャットを行い、メッセージを送受信します。
   - `server/users/usr_local_dev_bypass/chat_history.json` が生成され、会話履歴が正しく書き込まれていることを確認します。

3. **カスタムアセット・音声の保存確認**:
   - 設定画面などからマスコットのアバター画像や衣装画像を新規アップロード（ローカル画像選択など）します。
   - `mascots/users/usr_local_dev_bypass/<mascot_id>/` 配下に画像が保存されていることを確認します。
   - チャット会話を行い、生成された音声ファイルが `mascots/users/usr_local_dev_bypass/<mascot_id>/voices/<date_str>/` に保存されていることを確認します。
   - アプリ上でアップロードした画像や音声が正しく表示・再生されることを確認します。

---

### テスト項目 2: 異なるユーザー間の完全独立性の検証
内部ユーザーID（`id`）が異なる別ユーザーでアクセスした際に、データが干渉しないことを確認します。

1. **擬似ユーザーでの検証（デバッグ手法）**:
   - 動作確認用に一時的に `server/src/middlewares/auth-middleware.ts` のローカルバイパス部分 (`isLocal`) の設定内容を以下のように書き換えて、アプリで設定変更やチャットを行います。
     - ユーザーA想定: `req.user = { id: 'usr_test_a', role: 'user' }`
   - 次に、以下のように書き換えて再度アプリを操作します。
     - ユーザーB想定: `req.user = { id: 'usr_test_b', role: 'user' }`
2. **期待される結果**:
   - `server/users/usr_test_a/` と `server/users/usr_test_b/` の両方に別々の `config.json` および `chat_history.json` が作成されること。
   - `usr_test_b` で動作している間は、`usr_test_a` のチャット履歴や設定変更が一切反映されず、完全に独立していること。
   - `mascots/users/usr_test_a/` と `mascots/users/usr_test_b/` に画像アセットが隔離されて保存されること。

---

## 3. 確認後の片付け
動作確認で使用した `server/users/usr_test_a` や `server/users/usr_test_b` などのテスト用ディレクトリ、および `auth-middleware.ts` のデバッグ用コードは、本番環境へのデプロイ前に削除・元に戻してください。
