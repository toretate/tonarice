## 📋 フェーズ 2: 設定ファイルとアセット配信のサーバー移行

- `[x]` **2-1. 設定ファイルのサーバー連携 (API化)**

  - `[x]` プロジェクトルートの `mascots/` フォルダを `server/mascots/` に移行（またはシンボリックリンクなどの構成決定）
  - `[x]` サーバー側に、特定のキャラクターの画像アセット（服装、表情、ポーズ）を取得するファイル配信API（静的配信）を実装

  - `[x]` `MascotViewer.vue` におけるローカル画像ファイルの読み込み処理（`select-local-image` 等）を、サーバーAPI経由でのロードへ差し替え
- `[x]` **2-4. config.json 肥大化防止のための画像ファイル自動抽出・物理保存化**

  - `[x]` 各種表示コンポーネント（`MascotViewer.vue`、`MascotSettings.vue`、`ExpressionEditorModal.vue`）で静的相対パス画像URLを検知し、サーバーのホストをプリフィックスして美しくオンデマンド描画する仕組みを実装

## 📋 フェーズ 4: クライアントプロセスの極限軽量化と最適化

- `[x]` **4-1. ハイブリッド動作（サーバー/ローカル）の維持と最適化**
  - `[x]` サーバー非連携時のローカル（Electron経由）動作を担保するため、`electron/main.ts` の IPC ハンドラーは「ローカルフォールバック用」として綺麗に維持・整理し、不要なデバッグコード等を最適化。
- `[x]` **4-2. マルチクライアント接続の安全性検証**
  - `[x]` 複数クライアント（デスクトップ窓や設定画面など）が同時接続しても、アセットが `mascotId` ごとに完全に分離されたディレクトリ（`server/mascots/[mascotId]/`）に物理保存されるため、並行保存時も競合・破損が発生しないことを設計保証。
