# [実装計画] 画像トリミングのリセットおよび元画像復元機能の実装

この計画書は、表情画像の切り出し（トリミング）UIにおいて、変な範囲で切り抜かれてしまった画像を「いつでも元の全体画像」に戻すことができる仕組み、および元の全体画像から再トリミングを行えるようにする復元機能の実装計画です。

## 解決する問題とアプローチ
現在、画像のトリミングを行うと、表情スロットのアセット画像 (`path`) が切り抜かれたあとの `base64` データで上書きされてしまいます。
このため、一度トリミングを完了してしまうと：
1. 「切り抜き直す」を押した際、すでに切り抜かれた後の狭い画像しか読み込まれず、再度広い範囲を選択することができない。
2. 「切り抜き解除（元画像全体）」を押しても、すでに切り抜かれた後の画像しか読み込まれていないため、元の全体画像に戻すことができない。

この問題を根本的に解決するため、表情アセットデータに元の画像ソースを保持する `originalPath` フィールドを導入し、トリミングUIおよび設定画面と連動させます。

## ユーザー確認事項
> [!NOTE]
> すでにトリミングUI (`ImageCropModal.vue`) には「切り抜き解除 (元画像全体)」および「範囲リセット」のボタンが実装されていますが、データ構造上「元画像」が引き渡されていなかったため動作していませんでした。今回の修正により、これらのボタンが期待通り完全に機能するようになります。

## 提案する変更

### 1. データ構造の拡張

#### [MODIFY] [MascotSettings.vue](file:///c:/workspace/workspace-win/DesktopAiMascot/src/components/settings/MascotSettings.vue)
- `MascotAsset` インターフェースに `originalPath?: string;` を追加。
- `handleCropCurrent` で `slot.originalPath` が存在する場合はそれを `cropImageSrc.value` にセットするように変更。
- `handleCropNew` で対象のスロット情報（`slot`）を引数で受け取るようにし、`selectedCropExpression.value` にバインド。
- `handleCropDone` で切り抜かれた画像を保存する際、`targetSlot.originalPath = cropImageSrc.value;` を設定して、元の画像ソースを永続化。

#### [MODIFY] [ExpressionEditorModal.vue](file:///c:/workspace/workspace-win/DesktopAiMascot/src/components/settings/ExpressionEditorModal.vue)
- `MascotAsset` インターフェースに `originalPath?: string;` を追加。
- 「画像から切り出し」ボタンのクリック時に `emit('crop-new', selectedModalExpression)` とし、現在選択中の表情スロットを渡すように変更。
- 表情スプライトの拡大率（Scale）の最大値を `2.5` から `2.0` に変更。
- 拡大率表示バッジ（紫色）の右側に、0.01刻みで微調整が可能な上下の山形（chevron）コントロールを設置。

## 検証計画

### 手動確認
1. 設定画面を開き、表情エディタを起動する。
2. 任意の表情（例: 「通常」や「怒り」）に対して「画像から切り出し」を選択し、ローカル画像からトリミングを行って保存する。
3. 保存された表情に対して「切り抜き直す」をクリックした際、**切り抜かれる前の元の全体画像**がトリミングUIに読み込まれることを確認する。
4. トリミングUI内の「切り抜き解除 (元画像全体)」ボタンをクリックし、表情画像が元の全体画像に一発で復元されることを確認する。
5. 再度「切り抜き直す」から変形し直せることを確認する。
6. 表情エディタ上で拡大率スライダーの最大値が `2.0` に制限されていることを確認する。
7. 拡大率表示の横にある上下山形ボタンをクリックしたとき、値が `0.01` 刻みで正確に加減され、プレビューがリアルタイムに連動することを確認する。
