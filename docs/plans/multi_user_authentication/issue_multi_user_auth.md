# Issue: 将来的なマルチユーザー対応のための認証の仕組みの設計とプロトタイプ実装

## 概要
現在、Desktop AI Mascotのサーバーはローカル環境（シングルユーザー）での稼働を前提としており、認証機構が存在しません。
将来的に複数のユーザーが同一のサーバーを共有し、それぞれの設定やマスコットデータを安全に管理・利用（マルチユーザー対応）できるようにするため、認証の仕組み（JWTおよびトークンベースの認証）を設計し、プロトタイプを導入します。

## 要求仕様
1. **ユーザー概念の導入**:
   - 各要求（REST API および WebSocket）がどのユーザーによるものかを識別できるようにする。
2. **トークンベース認証 (JWT)**:
   - REST API 向けに JWT (JSON Web Token) による認証を導入し、有効なトークンを持つリクエストのみ処理するように保護する。
3. **WebSocket の認証**:
   - WebSocket 接続開始時（または接続直後のメッセージ送信）に認証トークンを検証し、接続元ユーザーを特定・固定できるようにする。
4. **セキュリティの考慮**:
   - トークンの署名検証および有効期限管理。
