# マスコット追加・編集UIの統合と動的反映の実装計画 (追加要件: ローカル画像インポート)

本計画は、ユーザーからの「ローカルの画像を登録する方法がない」というご要望に応え、PC上の画像ファイル（PNG、JPG、SVG等）を選択・ロードし、マスコットの各アセット（アバター、服装、表情、ポーズ）としてアプリ全体（設定、デスクトップ表示）に美しく反映させるための追加修正計画です。

---

## 1. 現状分析と課題

1. **アセット追加時の制限**:
    - 現在、アセット追加ダイアログ（`SettingsWindow.vue`）では「絵文字」の登録しか考慮されていません。
    - ローカルPCの画像ファイルを選択するファイルダイアログや、それをマスコットアセットとして保存・描画する仕組みが不足しています。

2. **レンダラー（Vue）でのローカルファイル表示のセキュリティ制限**:
    - Electron環境でローカルの絶対ファイルパス（`C:\path\to\image.png`）をそのまま `<img>` の `src` に指定すると、セキュリティポリシー（CORS/CSP）や `file://` プロトコルの制限に引っかかり描画できない場合があります。

---

## 提案する解決策

### 1) メインプロセスに画像ファイル選択＆Base64変換ハンドラーを追加 (`electron/main.ts`)
- `select-local-image` というIPCハンドラーを実装します。
- `dialog.showOpenDialog` を使用してローカルPCのファイルダイアログを開き、画像ファイル（`*.png`, `*.jpg`, `*.jpeg`, `*.gif`, `*.webp`, `*.svg`）を選択可能にします。
- 選択された画像をメインプロセス側で即座に読み込み、**Base64データURL** (`data:image/png;base64,...`) に変換してレンダラープロセス（Vue）に返します。これにより、セキュリティ制限を完全に回避し、あらゆるローカル画像を即座に安定描画できます。

### 2) プリロードスクリプトへのAPI追加 (`electron/preload.ts`)
- `window.electronAPI.selectLocalImage()` を公開し、Vueから簡単にローカル画像選択ダイアログを起動できるようにします。

### 3) アセット追加ダイアログでの画像選択UIの実装 (`SettingsWindow.vue`)
- アセット追加モーダルに「ローカル画像を選択」ボタンを追加します。
- ボタンクリック時に上記APIを呼び出し、取得したBase64データURLをアセットの `path` に設定します。
- 同時に、アセット名が空の場合は、選択したファイルのファイル名（拡張子なし）を自動で「アセット名」に補完する親切設計にします。

### 4) レンダラー全体での「画像アセット」描画の対応 (`SettingsWindow.vue`, `MascotViewer.vue`)
- アセット의 `path` が絵文字（長さが極めて短いテキスト）か、それとも画像データ（`data:image/` や `http`, `C:\` 等で始まる文字列）かを判定します。
- 画像データの場合は、`<span>` で絵文字を描画する代わりに、`<img>` タグを生成してアセットを完全に描画します。
- 設定画面の「合成プレビュー」および「デスクトップマスコット表示」の双方で、絵文字と画像ファイルのどちらが登録されていても自動で完璧に重ね合わせて描画されるようにCSSレイアウトとテンプレートを修正します。

## 4. 動作確認・検証計画

### 4.1. 自動検証（ビルドチェック）
- `npm.cmd run build` を実行し、TypeScriptのコンパイルエラーが無いことを検証します。

### 4.2. 手動動作テスト
1. **ローカル画像の登録**:
    - 設定画面でマスコットの服装や表情アセットの「追加」ボタンを押し、「ローカル画像を選択」をクリック。
    - PC上の任意のPNG/JPGファイルを選択し、プレビューにその画像がアセットとしてカード登録され、アセット名がファイル名に自動補完されることを確認。
2. **合成プレビューとデスクトップ表示の連動**:
    - 登録した画像アセット（例: 服装のPNG画像）を選択した際、設定画面のプレビューおよびデスクトップ上のマスコットの重ね合わせ表示で、その画像が美しく切り替わって表示されることを確認。
