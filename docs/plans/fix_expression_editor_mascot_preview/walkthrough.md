# 表情エディタのマスコットプレビュー不具合修正 修正確認資料 (Walkthrough)

表情エディタにおいて、キャラクターの「全身像（立ち絵）」自体をUIから登録・管理する画面が欠落していたため、マスコットの全身像背景が表示されず、表情パーツの位置合わせが行えなかった不具合を修正しました。
また、立ち絵が綺麗に中央に描画されるようになった一方で、表情パーツ画像が全身像画像の背面（背後）に隠れてしまい、位置合わせが行えなくなっていた `z-index` 重ね合わせ順の不具合を特定し、完全に解消しました。

## 変更内容

### 1. 表情パーツ（表情レイヤー）の手前重ね合わせの保証 (z-indexの修正)

#### [MODIFY] [SettingsWindow.vue](file:///c:/workspace/workspace-win/DesktopAiMascot/src/components/settings/SettingsWindow.vue)

- **`preview-layer-img` および `preview-layer` への `z-index: 10` の追加**:
  表情エディタのプレビューエリアにおいて、表情パーツ（女の子の顔）を描画するクラスである `.preview-layer-img` （画像による表情パーツ）と `.preview-layer` （絵文字等による表情パーツ）に `z-index: 10` を追加しました。
  これにより、背景となる立ち絵全身像（`.preview-full-img` および `.preview-base-avatar` の `z-index: 1`）の手前（前面）に**表情パーツが 100% 確実に重ね合わされて表示される**ようなり、ユーザーが全身像の顔の位置に合わせて表情のオフセットやスケールを正確に調整できるようになりました。

### 2. 全身像プレビュー用CSSレイアウトの修正・中央配置の保証

- **`preview-full-img` クラスへの絶対配置および最大サイズ指定の追加**:
  プレビューエリアの背景画像を描画する `.preview-full-img` クラスに `top: 0; left: 0; max-width: 100%; max-height: 100%;` の指定を追加しました。
  これにより、どんな高解像度な立ち絵画像であっても、親コンテナであるプレビューカードエリア（`420px x 420px`）から絶対に飛び出さず、その中にぴったりフィットしてアスペクト比を維持したまま中央に収まるよう制限しました。

### 3. 表情エディタ背景プレビューにおける画像アセット優先解決ロジックの実装

- **画像アセット（`data:image/`）の最優先描画ロジック**:
  表情エディタの背景解決テンプレートを修正し、マスコットデータに `activePose` が登録されている場合（デフォルトの絵文字 `🧍` など）であっても、ユーザーが登録した `activeOutfit`（衣装アセット）に画像パス（`data:image/` から始まる立ち絵画像）が存在するならば、**その画像を最優先でコンテナ背景に描画する**ように修正しました。

### 4. 立ち絵データのリアルタイム完全自動同期 (computed化)

- **`activeOutfit` / `activePose` の computed 化**:
  `activeOutfit` および `activePose` の定義を `ref` から `computed` に変更しました。これにより、設定画面で立ち絵を追加したりメインの立ち絵を設定したりした際、表情エディタモーダル内の背景全身像（`activeOutfit`）が、データの変更を検知して自動的かつ確実に最新 of 画像に同期・再描画されるようになりました。
