# 音声合成の文節分割と順次再生（ストリーミング再生）の実装計画

AIの返答が長い場合に音声が生成され始めるまでの遅延（レイテンシ）を削減するため、返答テキストを文節（文）ごとに分割し、並行して音声合成を行いながら、準備ができた音声から順次再生する仕組みを導入します。

## ユーザーレビュー要求事項

> [!NOTE]
> - テキストの分割は「。！？\n」などの句読点を基準に行います。
> - 音声の連続再生中（発話中）にユーザーが新たにメッセージを送信した場合、再生中の音声を即座に停止し、再生キューをクリアした上で新しい会話の音声再生を開始します。

## 変更内容の提案

音声再生を管理するプレイリストクラスをフロントエンドに導入し、非サーバーモードとサーバーモードの双方でテキスト分割・並行合成を行います。

---

### フロントエンド共通プレイヤー

#### [NEW] [AudioPlaylist.ts](file:///c:/workspace/workspace-win/DesktopAiMascot/ui/src/utils/AudioPlaylist.ts)
- 音声再生のキューを管理するシンプルなプレイリストクラスを作成します。
- `push(base64Audio: string)`: 音声データをキューに追加し、再生中でなければ順次再生を開始します。
- `stop()`: 現在再生中の音声を停止し、キューをクリアします。
- 再生中および終了時のステータスをコールバックを通じて通知し、マスコットの `speaking` フラグを制御します。

---

### クライアント側（非サーバーモード）

#### [MODIFY] [ChatPanel.vue](file:///c:/workspace/workspace-win/DesktopAiMascot/ui/src/components/ChatPanel.vue)
- 新しく作成する `AudioPlaylist` インスタンスを保持します。
- `sendMessage` 内で、返答テキストを句読点で分割します。
- 分割した各文に対する音声合成（`synthesizeVoicevox`）の Promise を並行して開始します。
- 順番を維持したまま各 Promise の完了を待ち（`await`）、完了したものから `AudioPlaylist` に追加していきます。
- 新たにメッセージを送信した際、以前の再生を停止するために `playlist.stop()` を呼び出します。

---

### サーバー側（サーバーモード：WebSocket）

#### [MODIFY] [websocket.ts](file:///c:/workspace/workspace-win/DesktopAiMascot/server/src/routes/websocket.ts)
- `speechText` を句読点等で分割します。
- 各文の `VoiceAiService.synthesize` 呼び出しを並行して実行します。
- 順番通りに Promise の完了を待ち、完了した文から順次 `chat-audio` イベントでクライアントにプッシュ送信します。
- フロントエンド側（`ChatPanel.vue`）の WebSocket レシーバーは、送られてきた `chat-audio` データをそのまま `AudioPlaylist` に追加します。

## 検証計画

### 自動ビルドとテスト
- `npm run build` が正常に通ることを確認します。
- フロントエンドの既存テストが破損していないことを確認します。

### 手動確認
- アプリケーションを起動し、長文の返答を生成させます。
- テキストの最初の文の表示後、素早く音声の再生が開始されることを確認します。
- 1つ目の文の再生中に2つ目の文が裏で合成され、1つ目が終わるとシームレスに2つ目の再生が始まることを確認します。
- 音声再生中に新たなメッセージを送信した際、再生が即座に中断され、新しい発話に切り替わることを確認します。
