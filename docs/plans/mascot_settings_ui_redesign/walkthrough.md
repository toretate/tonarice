## 変更内容

### 1. ウィンドウ全体の拡張とスケール対応

### 2. 左ナビゲーション（カテゴリ選択）のプレミアムスタイリング
- **[MODIFY] [SettingsWindow.tscn](file:///c:/workspace/workspace-win/DesktopAiMascot/ui/settings/SettingsWindow.tscn)**
  - 左ナビメニュー（`CategoryList`）の幅を指示書どおり `180px` に固定配置しました。
  - 選択中のカテゴリに対して、指示書に描かれた「**淡い青色背景**」かつ「**左端アクセント線（テーマカラー `#3478F6`、太さ `4px`）**」を適用しました。
  - ホバー時や選択時などの微細なアニメーションや透明度（`StyleBoxFlat` / `StyleBoxEmpty`）を適用し、ホバーフィードバックが直感的で alive なデザインを再現しました。

### 3. 3ペイン構成（リスト・プレビュー・詳細）の導入
- **[MODIFY] [MascotPropertyPage.tscn](file:///c:/workspace/workspace-win/DesktopAiMascot/ui/settings/pages/MascotPropertyPage.tscn)**
  - 画面を3つの論理的なペイン（左カラム：リスト、中央カラム：全身プレビュー、右カラム：詳細）に再編しました。
  
  1. **左カラム (マスコットカードリスト)**:
     - 縦スクロールのみを許可する `ScrollContainer` 内に、個別のカードを動的に配置します。
     - 下部に白背景の `Add` ボタン（角丸 `8px`、余白やボタン高さを指示書に最適化）を固定配置しました。
  
  2. **中央カラム (全身プレビューペイン)**:
     - 伸縮主体 (`size_flags_horizontal = 3`) とし、角丸 `8px` で枠線（`Color(0.88, 0.90, 0.92)`）を持つ上品な薄グレー背景の `PreviewPanel` を新設しました。
     - 内部にスクロールを可能にする `ScrollContainer` を入れ、その中に全身画像をアスペクト比維持で中央配置する `TextureRect`（`PreviewTexture`、アスペクト比維持：`Keep Aspect Centered`）を実装しました。
  
  3. **右カラム (詳細＆操作ペイン)**:
     - 横幅を `360px` 固定 (`custom_minimum_size = Vector2(360, 0)`) にし、縦一杯に広がる構成としました。
     - 上品な影（シャドウサイズ `8px`、オフセット `(0, 4)`）を持つ角丸 `8px` の白背景カード型パネル（`InfoPanel`）を配置しました。
     - 各項目を整理し、上部に `Current Voice Settings` およびマスコット名（大文字・太字）、中間に区切り線、下部に `Voice Style` と薄い小文字の `voice settings` を配置する構成に再構築しました。

### 4. アクションボタンの再設計と角丸の統一 (`8px`)
- **[MODIFY] [MascotPropertyPage.tscn](file:///c:/workspace/workspace-win/DesktopAiMascot/ui/settings/pages/MascotPropertyPage.tscn)**
  - すべてのパネル、カード、ボタンの角丸を指示書通り `8px` に統一しました。
  - **プライマリボタン（Edit）**:
    - 主アクションとして、鮮やかなプライマリブルー（`#3478F6`、白文字、角丸 `8px`）の塗りつぶしスタイルへと一新し、左側に `📝` アイコンを美しく表示させました。
  - **セカンダリボタン（Generate Emotes, Remove Background）**:
    - アウトライン形式で、それぞれ指示書のカラーコードに即した薄いカラーのセカンダリスタイル（通常・ホバー・フォーカス）を完全に定義し、リデザインしました。

### 5. C# コードによる全身プレビュー画像の自動ロードと角丸調整
- **[MODIFY] [MascotPropertyPage.cs](file:///c:/workspace/workspace-win/DesktopAiMascot/ui/settings/pages/MascotPropertyPage.cs)**
  - 全身プレビュー用の `TextureRect` を `%PreviewTexture` としてバインドしました。
  - 動的に生成するリスト内のマスコットカードの角丸を、指示書に合わせて `12` から `8` へ変更しました。
  - マスコット選択切り替え時（`UpdateLabels()` 内）に、選択されたマスコットモデルの正面画像（`GetFrontImage()`）または代表画像（`GetPrimaryImage()`）の画像パスを検出し、自動でプレビュー領域にロード・アスペクト比を維持して描画するロジックを実装しました。画像が無い場合は安全に非表示にします。

## 検証結果

### 1. コンパイル検証
- 修正完了後、`dotnet build` を実行し、**エラー 0** でビルドが正常に成功することを確認しました。C# スクリプトのシグネチャ調整やバインディングも問題ありません。

### 2. 動作の確認（手動検証手順）
以下の手順で、再設計された 3ペインのプレミアム UI をご確認ください。

1. **プロジェクトのビルド**:
   - `dotnet build` を実行して、ビルドが完了していることを確認します。
2. **Godot エディタまたはデバッグ実行**:
   - Godot エディタ、または `Main` シーンから本設定ウィンドウを開きます。
3. **リサイズ確認**:
   - ウィンドウを `1024x768` からドラッグしてリサイズした際、左カラムの幅（180px）と右詳細カラムの幅（360px）が固定されたまま、**中央のプレビューペインのみが伸縮（Expand Fill）**し、中のマスコット画像がアスペクト比を維持したまま綺麗に拡大縮小することを確認します。
4. **マスコット切り替えの連動**:
   - 左カラムのカード（角丸 8px）をクリックすると、中央プレビューのマスコット全身画像が瞬時に切り替わり、右カラムの `miyako` などの詳細ラベルが動的に書き換わることを確認します。
5. **ボタンのホバー・クリック**:
   - 右詳細ペイン下部の青塗りつぶし `Edit` ボタン、およびアウトライン形式のセカンダリボタン群が、カーソルを合わせた際に滑らかに反応することを確認します。
