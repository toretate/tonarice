# 実装計画: PixiJSの導入とマスコット描画の移行

本計画書は、Desktop AI Mascot の描画エンジンを現在の HTML/CSS (DOM) ベースの重ね合わせから、WebGLベースの2Dレンダリングエンジンである **PixiJS** に移行するための手順と設計を示します。これにより、将来的な「瞬き」「口パク」などのアニメーション制御の高速化と、チラつきのない表現を実現します。

---

## 1. 移行の目的
*   **アニメーション表現の布石**: 目や口のスプライトアニメーション、リアルタイムなリップシンク（口パク）をカクつきや遅延なく実行可能にする。
*   **描画パフォーマンスの向上**: WebGLによるハードウェアアクセラレーションを活用し、複数レイヤーの重ね合わせやブレンド処理の負荷を低減する。
*   **アセット管理の最適化**: テクスチャ（スプライト）のロードとメモリ管理を一元化し、画像切り替え時のホワイトアウト（デコード遅延によるチラつき）を防ぐ。

---

## 2. 移行設計

### 2.1. 描画プロセスの変更
```
[従来: DOMアプローチ]
Vue.js (template) ──> <img> (ポーズ/服装) ──> CSS Transform で位置合わせ
                 ──> <img> (表情レイヤー) ──> CSS Transform で位置合わせ
                 
[移行後: PixiJSアプローチ]
Vue.js (template) ──> <canvas> (マウント先)
                             │
                             ▼ (PIXI.Applicationによる制御)
PIXI.Container (マスコット全体)
  ├── PIXI.Sprite (ベース/ポーズ/服装)
  └── PIXI.Sprite (表情レイヤー) ──> PixiJS Transform (x, y, scale, rotation)
```

### 2.2. 主要な設計変更点
*   **描画キャンバスの追加**: [MascotViewer.vue](file:///C:/workspace/workspace-win/DesktopAiMascot/ui/src/components/MascotViewer.vue) の `<template>` に描画用の `<canvas>` を配置し、Vueの `onMounted` ライフサイクルで `PIXI.Application` を初期化・マウントします。
*   **テクスチャのロードと切り替え**: 画像パスの変更（感情変化や服装変更）を監視（`watch`）し、PixiJSの `Assets` API（または `Texture.from`）を用いて非同期にテクスチャをロードしてスプライトへ適用します。
*   **位置合わせ（Transform）の適用**: 設定画面やプレビュー状態から送られてくるオフセット（X/Y）、スケール、回転角のパラメータを、PixiJSの `PIXI.Sprite` の `position`, `scale`, `rotation`（ラジアン変換）に同期させます。
*   **透明ピクセル判定 (境界計算)**: 現状の `getNonTransparentBounds` (一時Canvasを生成してgetImageData) のロジックをリファクタリングし、PixiJSのテクスチャからピクセルデータを効率的に取得するか、描画領域のバウンディングボックス計算を最適化します。

## 3. 実装ステップ

### フェーズ 1: 環境構築とボイラープレート実装
1.  `ui` プロジェクトに `pixi.js` をインストールする。
2.  `MascotViewer.vue` にて、Vueライフサイクルと連動した PixiJS の初期化とクリーンアップ（`app.destroy`）の基本構造を構築する。

### フェーズ 2: 静的レンダリングと位置合わせの移行
1.  ポーズ・服装・ベース画像の読み込みと PixiJS スプライトでの重ね合わせ描画を実装する。
2.  表情（Expression）レイヤーのスプライト重ね合わせと、CSSから移行した `transform`（位置調整値）の適用。
3.  プレビュー状態（`previewState`）の変更が PixiJS 側にリアルタイムに反映される同期ロジックを実装する。

### フェーズ 3: インタラクションと特殊処理の対応
1.  ウィンドウのドラッグ移動・右クリックメニュー起動が従来通り動作するよう、PixiJSキャンバス上でのイベント伝播（ポインターイベント）を調整する。
2.  クリック透過判定用の不透明境界計算（`updateCharacterBounds`）の移植。

### フェーズ 4: テストと最適化
1.  画像切り替え時のチラつきや、メモリリーク（テクスチャの破棄漏れ）がないかを検証する。
2.  コンパクトモードや設定変更時のスケーリング挙動の整合性を確認する。
