# タスクリスト: PixiJS移行タスク

本タスクリストは、マスコット描画エンジンを PixiJS に移行するための詳細な作業項目です。

## [x] フェーズ 1: 環境構築と準備
- [x] `ui` ディレクトリでの `pixi.js` パッケージのインストール
- [x] `MascotViewer.vue` にて PixiJS のインポートと Canvas 要素の準備
- [x] Vueの `onMounted` / `onUnmounted` における `PIXI.Application` の初期化と破棄処理の実装

## [x] フェーズ 2: スプライト描画の実装
- [x] ベースアバター/ポーズ/服装テクスチャを非同期ロードして描画する `MascotContainer` の作成
- [x] 表情（Expression）レイヤーのスプライト重ね合わせ処理の実装
- [x] 表情のオフセット（X/Y）、スケール、回転角を PixiJS 側にリアルタイム同期する処理の構築
- [x] `previewState`（編集画面でのリアルタイムプレビュー）の変更を PixiJS 側に反映させるロジック of 移植

## [x] フェーズ 3: インタラクションとシステム連携の移行
- [x] マスコットのドラッグイベント（Electron側へのドラッグ要求）とクリックイベント（チャットパネル開閉）の動作維持
- [x] 透明ピクセル境界検出（`getNonTransparentBounds`）の移植と `updateCharacterBounds` によるクリック透過領域の更新処理の移植
- [x] マスコット全体のスケール（`totalMascotScale`）の変更を PixiJS 描画に適用する処理の実装 (CSS/DOMによるスケーリング制御との併用で完了)

## [ ] フェーズ 4: アニメーション機能の準備（将来用ロードマップ）
- [ ] 瞬きや口パクのためのスプライト切り替え式アニメーション（`AnimatedSprite`）の設計・雛形作成
- [ ] 音声再生状態（`isSpeaking`）に応じたアニメーションループのトリガー検証

## [ ] フェーズ 5: 動作検証と調整
- [ ] 画像切り替え時のチラつきがないことの確認
- [ ] 設定画面での位置合わせ操作が正しくプレビューに反映されることの確認
- [ ] メモリリーク（不要になったテクスチャが破棄されずに残る現象）の発生確認とクリーンアップ処理の検証
