# Geminiへの仕様・設計相談

Geminiから独立した意見を得て、Codexが根拠を確認し、採用判断をユーザーへ提示する。ファイルは変更しない。

## 手順

1. 相談したい判断、前提、制約、候補案をCodex自身で整理する。
2. 必要最小限の仕様書や設計資料だけを対象にする。機密情報、認証情報、無関係なファイルを含めない。
3. Antigravityのラッパーを `-Tier pro -Mode plan -Sandbox -Digest -Dir <repo>` で実行する。
4. Geminiへ、前提の妥当性、選択肢、利点・欠点、失敗条件、見落とし、推奨案、追加確認事項を要求する。ファイル編集、コマンドによる状態変更、コミット、外部送信を禁止する。
5. Geminiの回答を仕様とソースで照合し、事実と推測を区別する。
6. Codexの見解と一致点・相違点を整理し、最終的な推奨と未解決事項をユーザーへ返す。

## 実行例

```powershell
& "$env:USERPROFILE\.codex\skills\antigravity\scripts\agy-delegate.ps1" `
    -Tier pro -Mode plan -Sandbox -Digest -Dir . -Timeout 10m `
    "この仕様案について読み取り専用のセカンドオピニオンを提示してください。前提、代替案、トレードオフ、回帰・運用リスク、推奨案、追加で確認すべき事項を整理してください。ファイルは変更しないでください。"
```

## 出力

次の順序で簡潔に統合する。

1. 結論
2. GeminiとCodexの一致点
3. 相違点とその理由
4. 推奨案とトレードオフ
5. 未解決事項

## 安全条件

- 読み取り専用を維持し、実装やテストを開始しない。
- Geminiの回答を事実として無条件に採用しない。
- 外部通信の追加、ファイル変更、コミット、push、deployを行わない。
