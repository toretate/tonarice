# Codexへの安全なタスク委譲スキル (codex-delegate)

このスキルは、別のAIエージェント（Codexなど）に対して実装や修正を安全かつ正確に依頼するための「指示書（`fix_instructions_*.md`）」を生成するためのものです。
特に、機密情報の漏洩防止や、悪意あるコードの実行を防ぐためのセキュリティガードレールを自動で適用します。

## 1. 委譲指示書の生成手順

指示書を生成する際は、以下のステップを順に実行してください。

### ステップ 1: コンテキスト情報の収集と最小化
- **必要なファイルの特定**: 修正が必要なファイルのみを抽出し、絶対パス（またはプロジェクトルートからの相対パス）をリストアップします。
- **コンテキストの最小化**: 不要なファイルや、プロジェクト全体を丸ごと指示書に添付することは避けてください。修正箇所の周辺行（前後20〜30行程度）のみを提示します。

### ステップ 2: セキュリティマスキング（重要）
- **機密情報の検出**: ファイル内容や指示書テキスト内に、以下の機密情報が含まれていないか厳重に確認します。
    - APIキー、トークン、秘密鍵（`AI_KEY`, `TOKEN`, `SECRET`, `private_key` など）
    - パスワードや認証情報
    - 個人情報（メールアドレス、氏名など）
- **マスキングの適用**: 検出された機密情報は、すべて `[REDACTED]` または `********` に置換してください。絶対にそのまま指示書に書き出してはいけません。

### ステップ 3: テストコマンドと検証条件の整理
- **実行コマンドの制限**: Codexが実行すべきテストコマンド（例: `npx vitest`）を指定します。
- **RCE（リモートコード実行）対策**: 指示書に記載するコマンドは、事前に定義された安全なテストコマンドのみに制限し、任意のシェルスクリプトや外部からファイルをダウンロードするコマンド（`curl`, `wget` など）を含めないでください。

---

## 2. 指示書の標準フォーマット

生成する指示書は、以下のテンプレートに従って `docs/plans/` または `docs/specs/tool-use/` ディレクトリ配下に `fix_instructions_<タスク名>.md` として保存します。

```markdown
# 修正指示書（for Codex）: [タスク名]

## 1. 前提・禁止事項（セキュリティガードレール）
- **APIキーや環境変数の直接記述禁止**: コード内にAPIキーやトークンをハードコードしないでください。必要な場合は環境変数から取得するように実装してください。
- **外部通信の禁止**: テストや実行において、許可されていない外部ドメインへの通信を発生させないでください。
- **依存関係の追加制限**: `package.json` 等に新しい外部ライブラリを無断で追加しないでください。

## 2. 対象ファイルと修正内容
- **ファイル**: [対象ファイル名](file:///path/to/file)
- **修正の目的**: [簡単な説明]
- **具体的な変更イメージ（差分）**:
```diff
// 既存のコードと、変更後のイメージを記載
```

## 3. 検証手順
以下のコマンドを実行し、テストがすべて通過することを確認してください。
`[安全なテストコマンド]`
```

---

## 3. セキュリティチェックリスト（出力前確認）

指示書を書き出す直前に、必ず以下のチェックリストをエージェント自身で評価してください。

- [ ] 指示書内に実際のAPIキーやパスワードが露出していないか？
- [ ] 提示するコードブロックは必要最小限に絞られているか？
- [ ] 指示書内のテストコマンドに `|`, `;`, `&` などを用いた不要なコマンド結合や、外部スクリプトの実行が含まれていないか？
- [ ] 改行コードは `CRLF` に統一されているか？
